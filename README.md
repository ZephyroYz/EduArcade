# EduArcade

This README is available in English and Spanish.  
Este README está disponible en inglés y español.


## 🇬🇧 English Version

**EduArcade** is a web-based system for managing an educational institution.  
It allows you to manage users with different roles (administrator, teacher, and student), as well as an internal ticketing system for support.

The system interface is currently available in Spanish.

## Features
- Role-based user management
- Personalized dashboard depending on user type
- Ticket system for handling issues
- MVC-based structure using Laravel

## Stack
- Backend: Laravel 11 (PHP)
- Database: PostgreSQL
- Frontend: Blade + Vite
- Other: JavaScript, CSS

## Context
This project started in the 7th semester as part of a Software Engineering course and evolved until the 10th semester.

During this process, the project was also developed at a documentation level (requirements, roles, scope, and metrics), while my main responsibility was implementing the system.

The project was later presented at an academic expo-style event.

## Project Status
The project is fully functional. The UI and some validations can be improved, but the core system is complete.

## What I Learned
- Structuring real-world applications using Laravel
- Working with relational databases using PostgreSQL
- Applying separation of concerns (MVC)
- Taking a project from development to a deployable state
- Understanding how development connects with earlier stages like requirements and scope definition

## Notes
This project is not intended to be a production-ready product,  
but rather a solid base that demonstrates the structure of a real application.

## 🇪🇸 Versión en Español

**EduArcade** es un sistema web para la gestión de una institución educativa. Permite manejar usuarios con diferentes roles (administrador, docente y alumno), así como un sistema de tickets para soporte interno. 

La interfaz del sistema está disponible únicamente en español.

## ¿Qué hace? 
- Gestión de usuarios con roles
- Dashboard personalizado por tipo de usuario
- Sistema de tickets para incidencias
- Estructura basada en MVC con Laravel

## Stack
- Backend: Laravel 11 (PHP)
- Database: PostgreSQL
- Frontend: Blade + Vite
- Other: JavaScript, CSS

## Contexto
Este proyecto comenzó en el 7° cuatrimestre como parte de la materia de Ingeniería de Software y fue evolucionando hasta el 10° cuatrimestre, donde se trabajó su documentación, definición de roles, métricas y alcance. Mi participación fue como desarrollador principal, encargándome de la implementación completa del sistema. Posteriormente, el proyecto fue presentado en un evento de exposición académica. 

## Estado del proyecto 
El proyecto es completamente funcional. Puede mejorar en UI y algunas validaciones, pero la base del sistema está completa. 

## Lo que aprendí 
- Estructurar aplicaciones reales con Laravel
- Conectar proyectos con base de datos, usando PostgreSQL
- Separación de responsabilidades (MVC)
- Llevar a producción temporal un proyecto
- Entender cómo el desarrollo se relaciona con etapas previas como definición de requisitos y alcance

## Notas
El proyecto no busca ser un producto final listo para producción, 
sino una base sólida que demuestra la estructura de una aplicación real.
